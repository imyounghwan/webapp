## 개요

### 평가 시스템 구조
MGINE UI/UX 평가 시스템은 Nielsen의 10 Heuristics를 기반으로 **26개 세부 항목**으로 분해하여 평가합니다.

- **편의성 항목**: 21개 (Usability 중심)
- **디자인 항목**: 5개 (Visual Design 중심)

### 직접 선별 모드의 목적
사용자가 평가 대상 페이지를 직접 선택하면 **정확도가 15~20%p 상승**합니다. 이는 각 항목이 특정 페이지 유형에서만 의미 있게 평가되기 때문입니다.

### 페이지 유형별 평가 목적

#### 1️⃣ 메인 페이지 (필수)
- **목적**: 전체 사이트 구조와 첫인상 평가
- **평가 초점**: 메뉴 구조, 시각적 일관성, 핵심 정보 배치
- **영향 항목**: N7_1 (빠른 접근), 전체 디자인 항목

#### 2️⃣ 검색 결과 페이지 (권장)
- **목적**: 검색 기능 및 필터링 평가
- **평가 초점**: 검색 UX, 결과 표시, 정렬/필터
- **영향 항목**: N7_3 (검색/필터), N10_1 (도움말 가시성)

#### 3️⃣ 로그인 페이지 (권장)
- **목적**: 인증 흐름 및 폼 검증 평가
- **평가 초점**: 입력 검증, 오류 메시지, 보안
- **영향 항목**: N1_3, N3_1, N5_1/2/3, N9_2 (8개 항목)

#### 4️⃣ 게시판 목록 페이지 (선택)
- **목적**: 목록 UI 및 네비게이션 평가
- **평가 초점**: Breadcrumb, 페이징, 정렬
- **영향 항목**: N1_1 (Breadcrumb), N3_3 (네비게이션)

#### 5️⃣ 게시판 상세 페이지 (선택)
- **목적**: 콘텐츠 구조 및 계층 평가
- **평가 초점**: 헤딩 구조, 가독성, 댓글
- **영향 항목**: N1_1 (Breadcrumb), N2_2 (흐름)

#### 6️⃣ 폼 작성 페이지 (선택)
- **목적**: 입력 UX 및 검증 평가
- **평가 초점**: 실시간 검증, 제약 안내, 오류 복구
- **영향 항목**: N1_3, N3_1, N5_1/2/3, N9_2 (8개 항목)

#### 7️⃣ FAQ/도움말 페이지 (선택)
- **목적**: 도움말 접근성 및 문서화 평가
- **평가 초점**: 검색 기능, 카테고리, 명확성
- **영향 항목**: N7_3 (검색), N10_1/2 (도움말/문서화)

#### 8️⃣ 사이트맵/소개 페이지 (선택)
- **목적**: 전체 구조 파악 및 일관성 평가
- **평가 초점**: 정보 구조, 용어 일관성
- **영향 항목**: N1_1 (Breadcrumb), N4_2 (일관성)

#### 9️⃣ 연락처/문의 페이지 (선택)
- **목적**: 문의 폼 및 정보 제공 평가
- **평가 초점**: 폼 UX, 연락처 명확성
- **영향 항목**: N1_3, N3_1, N5_1/2/3, N9_2 (8개 항목)

#### 🔟 에러 페이지 (선택)
- **목적**: 오류 처리 및 복구 안내 평가
- **평가 초점**: 오류 메시지 명확성, 복구 방법
- **영향 항목**: N9_4 (오류 안내)

| 순위 | 페이지 | 이유 | 추가 커버 항목 |
|------|--------|------|----------------|
| 2위 | 🔐 로그인 또는 ✏️ 폼 작성 | 폼 관련 8개 항목 평가 | N1_3, N3_1, N5_1/2/3, N9_2 |
| 3위 | 🔍 검색 결과 또는 ❓ FAQ | 검색/도움말 2개 + Breadcrumb 2개 | N1_1, N6_3, N7_3, N10_1 |

#### **선택 페이지 (필요 시 추가)**
| 순위 | 페이지 | 추가 이유 |
|------|--------|-----------|
| 4위 | 📄 게시판 상세 | Breadcrumb 다양성, 콘텐츠 구조 |
| 5위 | ✉️ 연락처/문의 | 추가 폼 샘플 |
| 6위 | 📋 게시판 목록 | 네비게이션 다양성 |
| 7위 | 🗺️ 사이트맵 | 정보 구조 파악 |
| 8위 | ⚠️ 에러 페이지 | 오류 처리 평가 |

#### **전략 1: 항목별 커버리지 극대화**
```
목표: 각 항목이 최소 2개 이상의 페이지에서 평가되도록

예시:
- 폼 관련 (8개): 로그인 + 문의 (2개 페이지)
```

```
✅ 상호작용형: 로그인, 폼 작성, 문의
✅ 탐색형: 검색 결과, 게시판 목록, 사이트맵
```

```
  우선: 게시판 목록, 게시판 상세, 글쓰기, 프로필
```

### 백엔드: itemRelevance.ts

```typescript
/**
 * Nielsen 평가 항목별 영향 페이지 추적
 * 
 * 이 파일은 각 평가 항목이 어떤 페이지에서 평가되는지를 정의합니다.
 */

import type { HTMLStructure } from './htmlAnalyzer'

export interface ItemEvaluationResult {
  itemId: string
  relevantPages: string[]  // 이 항목 평가에 영향을 준 페이지들
  reason: string           // 왜 이 페이지들이 관련되는지
}

/**
 * 각 페이지에서 특정 항목의 영향도를 평가
 * 
 * @param pageResults 분석된 페이지 배열
 * @returns Map<itemId, relevantPages[]>
 */
export function evaluateItemRelevance(pageResults: any[]): Map<string, string[]> {
  const itemRelevance = new Map<string, string[]>()
  
  // 폼이 있는 페이지 필터링 (공통)
  const formPages = pageResults
    .filter(p => p.structure.forms.formCount > 0)
    .map(p => p.url)
  
  // 검색이 있는 페이지 필터링 (공통)
  const searchPages = pageResults
    .filter(p => p.structure.navigation.searchExists)
    .map(p => p.url)
  
  // ========================================
  // N1. 시스템 상태 가시성 (3개)
  // ========================================
  
  // N1.1 - Breadcrumb (서브 페이지에서만 의미있음)
  const breadcrumbPages = pageResults
    .filter(p => !p.isMainPage && p.structure.navigation.breadcrumbExists)
    .map(p => p.url)
  if (breadcrumbPages.length === 0) {
    // Breadcrumb이 없으면 서브 페이지들이 감점 원인
    itemRelevance.set('N1_1', pageResults.filter(p => !p.isMainPage).map(p => p.url))
  } else {
    itemRelevance.set('N1_1', breadcrumbPages)
  }
  
  // N1.2 - ARIA 레이블 (모든 페이지 평균)
  const ariaPages = pageResults
    .filter(p => p.structure.accessibility.ariaLabelCount > 0)
    .map(p => p.url)
  itemRelevance.set('N1_2', ariaPages.length > 0 ? ariaPages : pageResults.map(p => p.url))
  
  // N1.3 - 폼 검증 (폼이 있는 페이지)
  const validationPages = pageResults
    .filter(p => p.structure.forms.validationExists || p.structure.forms.formCount > 0)
    .map(p => p.url)
  itemRelevance.set('N1_3', validationPages.length > 0 ? validationPages : [pageResults[0].url])
  
  // ========================================
  // N2. 현실 세계와의 일치 (3개)
  // ========================================
  
  // N2.1 - lang 속성 (모든 페이지)
  const langPages = pageResults
    .filter(p => p.structure.accessibility.langAttribute)
    .map(p => p.url)
  itemRelevance.set('N2_1', langPages.length > 0 ? langPages : pageResults.map(p => p.url))
  
  // N2.2 - 헤딩 구조 (모든 페이지)
  itemRelevance.set('N2_2', pageResults.map(p => p.url))
  
  // N2.3 - 아이콘 (모든 페이지) - 디자인 항목
  itemRelevance.set('N2_3', pageResults.map(p => p.url))
  
  // ========================================
  // N3. 사용자 통제와 자유 (2개)
  // ========================================
  
  // N3.1 - 폼 (폼이 있는 페이지)
  itemRelevance.set('N3_1', formPages.length > 0 ? formPages : [pageResults[0].url])
  
  // N3.3 - 링크 (모든 페이지)
  itemRelevance.set('N3_3', pageResults.map(p => p.url))
  
  // ========================================
  // N4. 일관성과 표준 (3개)
  // ========================================
  
  // N4.1 - 시각적 일관성 (모든 페이지) - 디자인 항목
  itemRelevance.set('N4_1', pageResults.map(p => p.url))
  
  // N4.2 - 용어 일관성 (모든 페이지)
  itemRelevance.set('N4_2', pageResults.map(p => p.url))
  
  // N4.3 - 표준 준수 (모든 페이지)
  itemRelevance.set('N4_3', pageResults.map(p => p.url))
  
  // ========================================
  // N5. 오류 예방 (3개)
  // ========================================
  
  // N5.1, N5.2, N5.3 - 폼 관련 (폼이 있는 페이지)
  itemRelevance.set('N5_1', formPages.length > 0 ? formPages : [pageResults[0].url])
  itemRelevance.set('N5_2', formPages.length > 0 ? formPages : [pageResults[0].url])
  itemRelevance.set('N5_3', formPages.length > 0 ? formPages : [pageResults[0].url])
  
  // ========================================
  // N6. 인식보다 기억 (2개)
  // ========================================
  
  // N6.2 - 아이콘 (모든 페이지)
  itemRelevance.set('N6_2', pageResults.map(p => p.url))
  
  // N6.3 - Breadcrumb (서브 페이지)
  itemRelevance.set('N6_3', pageResults.filter(p => !p.isMainPage).map(p => p.url))
  
  // ========================================
  // N7. 유연성과 효율성 (3개)
  // ========================================
  
  // N7.1 - 메뉴 (주로 메인 페이지)
  itemRelevance.set('N7_1', [pageResults[0].url])
  
  // N7.2 - 맞춤설정 (모든 페이지)
  itemRelevance.set('N7_2', pageResults.map(p => p.url))
  
  // N7.3 - 검색 (검색이 있는 페이지)
  itemRelevance.set('N7_3', searchPages.length > 0 ? searchPages : pageResults.map(p => p.url))
  
  // ========================================
  // N8. 미니멀 디자인 (3개) - 전부 디자인 항목
  // ========================================
  
  itemRelevance.set('N8_1', pageResults.map(p => p.url))
  itemRelevance.set('N8_2', pageResults.map(p => p.url))
  itemRelevance.set('N8_3', pageResults.map(p => p.url))
  
  // ========================================
  // N9. 오류 인식과 복구 (2개)
  // ========================================
  
  // N9.2 - 오류 복구 (폼이 있는 페이지)
  itemRelevance.set('N9_2', formPages.length > 0 ? formPages : [pageResults[0].url])
  
  // N9.4 - 오류 메시지 (모든 페이지)
  itemRelevance.set('N9_4', pageResults.map(p => p.url))
  
  // ========================================
  // N10. 도움말과 문서 (2개)
  // ========================================
  
  // N10.1 - 도움말 (검색이 있는 페이지)
  itemRelevance.set('N10_1', searchPages.length > 0 ? searchPages : pageResults.map(p => p.url))
  
  // N10.2 - 문서화 (모든 페이지)
  itemRelevance.set('N10_2', pageResults.map(p => p.url))
  
  return itemRelevance
}
```

```typescript
// 실시간 URL 분석 API
app.post('/api/analyze', authMiddleware, async (c) => {
  try {
    const { url, urls, mode = 'mgine', usePuppeteer = false } = await c.req.json()

    // urls 배열이 제공된 경우 (사용자 직접 선별)
    if (urls && Array.isArray(urls) && urls.length > 0) {
      console.log('[Manual Selection] User provided', urls.length, 'URLs')
      
      // URL 유효성 검사
      const validUrls = urls.filter((u: string) => u && u.startsWith('http'))
      if (validUrls.length === 0) {
        return c.json({ error: 'No valid URLs provided' }, 400)
      }
      
      // 1. 사용자 제공 URL들을 분석
      let pageResults: any[] = []
      
      for (let i = 0; i < validUrls.length; i++) {
        const pageUrl = validUrls[i]
        const isMainPage = i === 0  // 첫 번째 URL을 메인 페이지로 간주
        
        console.log(`[${i + 1}/${validUrls.length}] Analyzing: ${pageUrl}`)
        
        try {
          const response = await fetch(pageUrl)
          if (!response.ok) continue
          
          const html = await response.text()
          const structure = analyzeHTML(html, pageUrl)
          
          pageResults.push({
            url: pageUrl,
            structure,
            isMainPage
          })
        } catch (error) {
          console.error(`Failed to analyze ${pageUrl}:`, error)
        }
      }
      
      if (pageResults.length === 0) {
        return c.json({ error: 'Failed to analyze any provided URLs' }, 500)
      }
      
      // 2. 종합 구조 생성
      const combinedStructure = combineMultipleStructures(pageResults)
      
      // 3. 항목별 영향 페이지 추적
      const itemRelevance = evaluateItemRelevance(pageResults)
      
      // 4. Nielsen 점수 계산
      const improvedScores = calculateImprovedNielsen(combinedStructure)
      const improvedDiagnoses = generateImprovedDiagnoses(
        combinedStructure, 
        improvedScores, 
        validUrls[0]
      )
      
      // 5. 편의성/디자인 항목 상세 생성 (피드백 적용)
      const convenience_items_detail: any[] = []
      const design_items_detail: any[] = []
      
      convenienceMapping.forEach((mapping, idx) => {
        const id = mapping.id
        const diagnosisKey = convenienceDiagnosisKeys[idx]
        const relevantPages = itemRelevance.get(id) || [validUrls[0]]
        
        // 기본 점수 계산
        let baseScore = (improvedScores as any)[diagnosisKey] || 0
        
        // 피드백 데이터 적용 (AI 학습 반영)
        const adjustedScore = applyFeedbackAdjustment(id, baseScore, validUrls[0])
        
        convenience_items_detail.push({
          item: desc?.name || mapping.key,
          item_id: id,
          category: '편의성',
          score: adjustedScore,  // 피드백 반영된 점수
          description: (improvedDiagnoses as any)[diagnosisKey]?.description || '...',
          recommendation: (improvedDiagnoses as any)[diagnosisKey]?.recommendation || '...',
          affected_pages: relevantPages  // 이 항목 평가에 영향을 준 페이지들
        })
      })
      
      // 디자인 항목도 동일하게 처리
      // ...
      
      // 6. 응답 반환
      return c.json({
        success: true,
        mode,
        analyzed_pages: pageResults.length,
        main_url: validUrls[0],
        sub_urls: validUrls.slice(1),
        predicted_score: {

          design: /* ... */
        },
        convenience_items: convenience_items_detail,
        design_items: design_items_detail,
        summary: generateEvaluationSummary(/* ... */)
      })
    }
    
    // 단일 URL 처리 (기존 로직)
    // ...
  } catch (error: any) {
    return c.json({ error: error.message }, 500)
  }
```

## 요약

### 핵심 포인트

1. **26개 항목 = 21개 편의성 + 5개 디자인**
   - 각 항목은 Nielsen 10 Heuristics 기반

2. **10가지 페이지 유형**
   - 필수 1개 (메인)
   - 권장 2개 (로그인/폼, 검색/FAQ)
   - 선택 7개

3. **영향도 분류**
   - 전체 페이지: 14개 항목
   - 폼 페이지: 8개 항목
   - 서브 페이지: 2개 항목
   - 검색 페이지: 2개 항목
   - 메인 페이지: 1개 항목

4. **최소 권장 조합**
   - 메인 + 로그인 + 검색 = 26개 항목 전부 평가
