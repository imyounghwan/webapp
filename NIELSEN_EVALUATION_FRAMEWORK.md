# AutoAnalyzer: 제이콥 닐슨 기반 UI/UX 평가 체계

## 📋 평가 체계 개요

**기존**: Q1~Q10 (10개 항목)  
**확장**: Nielsen 10원칙 기반 **25개 세부 평가 항목**

---

## 🎯 Nielsen 10가지 사용성 원칙 → 25개 세부 항목 매핑

### 1️⃣ Visibility of System Status (시스템 상태 가시성)
**기존 연결**: Q1 (목적 달성), Q7 (시각적 구분)

**세부 평가 항목**:
- **N1.1** 현재 페이지 위치 표시 (Breadcrumb, 활성 메뉴 하이라이트)
- **N1.2** 로딩 상태 표시 (Progress bar, Spinner)
- **N1.3** 작업 완료 피드백 (성공/실패 메시지)

**점수 산출**:
- Breadcrumb 존재 + 정확성 (40%)
- 로딩 인디케이터 유무 (30%)
- 사용자 액션 후 피드백 제공 (30%)

**구체적 진단 예시**:
```
✅ 4.5점: Breadcrumb가 모든 페이지에 존재하며, 로딩 시 명확한 인디케이터 제공
⚠️ 3.2점: Breadcrumb 없음, 로딩 상태 불명확 → 사용자가 현재 위치와 진행 상태를 파악하기 어려움
💡 개선안: 상단에 "홈 > 카테고리 > 현재페이지" 형태의 Breadcrumb 추가, 로딩 시 진행률 표시
```

---

### 2️⃣ Match Between System and Real World (시스템과 현실의 일치)
**기존 연결**: Q3 (전문성 없이 이해 가능), Q9 (보편적 구조)

**세부 평가 항목**:
- **N2.1** 사용자 친화적 용어 사용 (전문 용어 최소화)
- **N2.2** 논리적 정보 구조 (실제 업무 흐름 반영)
- **N2.3** 친숙한 아이콘 및 메타포 사용

**점수 산출**:
- 전문 용어 비율 (< 10%: 5점, 10~30%: 3점, >30%: 1점)
- 메뉴 구조의 직관성 (계층적 vs 혼재)
- 아이콘의 표준 준수 (Material Design, Font Awesome 등)

**구체적 진단 예시**:
```
✅ 4.7점: "신청하기", "조회하기" 등 일상 언어 사용, 메뉴가 사용자 관점으로 구성
⚠️ 2.8점: "대장조회", "통합민원" 등 행정 용어 남발 → 일반 시민 이해 어려움
💡 개선안: "내 신청 내역 보기", "문의하기"로 변경, 각 용어에 말풍선 설명 추가
```

---

### 3️⃣ User Control and Freedom (사용자 제어 및 자유도)
**기존 연결**: Q2 (이용 편리성), Q4 (메뉴 이동)

**세부 평가 항목**:
- **N3.1** 뒤로가기/취소 기능 제공
- **N3.2** 입력 중 임시 저장 기능
- **N3.3** 검색 기록 삭제 및 관리 기능

**점수 산출**:
- 모든 페이지에서 이전 단계 복귀 가능 (40%)
- 긴 폼에서 임시 저장 제공 (30%)
- 사용자가 자신의 데이터 제어 가능 (30%)

**구체적 진단 예시**:
```
✅ 4.8점: 뒤로가기 버튼 명확, 긴 신청서에 임시저장 제공, 검색 기록 삭제 가능
⚠️ 3.1점: 뒤로가기 시 입력 데이터 손실, 임시저장 없음 → 사용자가 실수 시 처음부터 다시 시작
💡 개선안: 브라우저 뒤로가기와 연동, 5분마다 자동 임시저장, "지우기" 버튼 추가
```

---

### 4️⃣ Consistency and Standards (일관성 및 표준)
**기존 연결**: Q8 (일관성), Q10 (색상 조화)

**세부 평가 항목**:
- **N4.1** 버튼 스타일 및 위치 일관성
- **N4.2** 색상 체계 일관성 (Primary, Secondary, Alert)
- **N4.3** 타이포그래피 일관성 (제목, 본문, 캡션)

**점수 산출**:
- 동일 기능 버튼의 스타일 통일도 (40%)
- 브랜드 컬러 사용 일관성 (30%)
- 글꼴 크기/굵기의 계층성 (30%)

**구체적 진단 예시**:
```
✅ 4.6점: 모든 주요 버튼이 동일한 파란색, 제목은 항상 24px 굵게
⚠️ 2.9점: 페이지마다 버튼 색상 다름 (파란색, 초록색, 빨간색 혼재) → 사용자 혼란
💡 개선안: 디자인 시스템 정의 (Primary: #0066CC, Danger: #CC0000), 모든 페이지에 적용
```

---

### 5️⃣ Error Prevention (오류 예방)
**기존 연결**: Q5 (경고 메시지)

**세부 평가 항목**:
- **N5.1** 입력 형식 가이드 제공 (예: 전화번호 010-0000-0000)
- **N5.2** 실시간 유효성 검사 (입력 중 오류 알림)
- **N5.3** 중요 작업 전 확인 메시지 (삭제, 제출 등)

**점수 산출**:
- 입력 필드에 placeholder/helper text 제공 (30%)
- 실시간 validation (30%)
- 위험 작업에 confirmation dialog (40%)

**구체적 진단 예시**:
```
✅ 4.9점: 전화번호 형식 자동 표시, 틀린 입력 시 즉시 빨간색 표시, 삭제 시 "정말 삭제하시겠습니까?" 확인
⚠️ 3.0점: 제출 후에야 오류 알림, 삭제 버튼 클릭 즉시 삭제 → 사용자 실수로 데이터 손실 위험
💡 개선안: 입력 필드 아래 "예: 010-1234-5678" 표시, 포커스 아웃 시 검사, 중요 버튼에 2단계 확인
```

---

### 6️⃣ Recognition Rather Than Recall (기억보다 인식)
**기존 연결**: Q6 (친절한 설명), Q7 (시각적 구분)

**세부 평가 항목**:
- **N6.1** 최근 검색어/이용 서비스 표시
- **N6.2** 자동완성 기능 제공
- **N6.3** 시각적 힌트 및 툴팁

**점수 산출**:
- 검색창에 최근 검색어 표시 (30%)
- 입력 시 자동완성 제안 (30%)
- 아이콘/버튼에 hover 시 설명 표시 (40%)

**구체적 진단 예시**:
```
✅ 4.7점: 검색 시 최근 검색어 자동 표시, 입력 중 연관 검색어 제안, 아이콘에 마우스 올리면 "출력" 등 설명
⚠️ 3.3점: 검색어를 매번 새로 입력해야 함, 아이콘만 있고 설명 없음 → 사용자가 기능 파악 어려움
💡 개선안: 최근 5개 검색어 표시, Google 스타일 자동완성, 모든 아이콘에 title 속성 추가
```

---

### 7️⃣ Flexibility and Efficiency of Use (유연성 및 효율성)
**기존 연결**: Q2 (이용 편리성), Q4 (메뉴 이동)

**세부 평가 항목**:
- **N7.1** 단축키 제공 (Ctrl+S 저장 등)
- **N7.2** 자주 찾는 메뉴 즐겨찾기 기능
- **N7.3** 고급 검색 옵션 (필터, 정렬)

**점수 산출**:
- 키보드 단축키 지원 (20%)
- 사용자 맞춤 메뉴 설정 가능 (30%)
- 검색 필터 다양성 (50%)

**구체적 진단 예시**:
```
✅ 4.5점: Enter로 검색, Tab으로 필드 이동, 상단에 "자주 찾는 메뉴" 커스터마이징 가능
⚠️ 2.7점: 마우스 클릭만 가능, 검색 시 필터 없음 → 파워유저가 비효율적으로 작업
💡 개선안: 주요 기능에 단축키 안내, "즐겨찾기 추가" 버튼, 날짜/카테고리 필터 제공
```

---

### 8️⃣ Aesthetic and Minimalist Design (심미적이고 미니멀한 디자인)
**기존 연결**: Q7 (시각적 구분), Q10 (색상 조화)

**세부 평가 항목**:
- **N8.1** 불필요한 정보 제거 (정보 밀도 적정성)
- **N8.2** 여백 및 그리드 시스템 활용
- **N8.3** 시각적 계층 구조 명확성

**점수 산출**:
- 한 화면에 표시되는 정보량 (적정: 5점, 과다: 1점)
- 여백 사용 적절성 (40%)
- 중요 정보 강조 효과 (크기, 색상, 위치) (40%)

**구체적 진단 예시**:
```
✅ 4.8점: 주요 정보만 표시, 충분한 여백, 제목과 본문이 명확히 구분
⚠️ 2.5점: 한 화면에 너무 많은 정보, 여백 없이 빽빽함 → 사용자가 집중할 포인트 찾기 어려움
💡 개선안: 정보를 탭/아코디언으로 숨김, 여백 20px 이상 확보, 주요 CTA 버튼만 강조
```

---

### 9️⃣ Help Users Recognize, Diagnose, and Recover from Errors (오류 인식, 진단, 복구 지원)
**기존 연결**: Q5 (경고 메시지), Q6 (친절한 설명)

**세부 평가 항목**:
- **N9.1** 명확한 오류 메시지 (기술 용어 회피)
- **N9.2** 해결 방법 제시
- **N9.3** 오류 발생 위치 표시

**점수 산출**:
- 오류 메시지의 명확성 (40%)
- 해결책 포함 여부 (40%)
- 오류 필드 시각적 강조 (20%)

**구체적 진단 예시**:
```
✅ 4.9점: "이메일 형식이 올바르지 않습니다. 예: user@example.com" → 해당 필드 빨간색 테두리
⚠️ 2.4점: "Error 500: Internal Server Error" → 사용자가 무엇을 해야 할지 모름
💡 개선안: "일시적 오류가 발생했습니다. 잠시 후 다시 시도해주세요." 오류 필드 scroll-into-view
```

### 🔟 Help and Documentation (도움말 및 문서)
**기존 연결**: Q6 (친절한 설명), Q3 (이해 가능성)

**세부 평가 항목**:
- **N10.1** FAQ/도움말 페이지 존재 및 접근성
- **N10.2** 맥락 기반 도움말 (Contextual Help)
- **N10.3** 튜토리얼/가이드 제공

**점수 산출**:
- FAQ 페이지 존재 + 검색 기능 (30%)
- 각 페이지에 물음표 아이콘 → 도움말 팝업 (40%)
- 첫 방문자를 위한 온보딩 (30%)

**구체적 진단 예시**:
```
✅ 4.7점: 우측 하단 "도움말" 버튼, 각 입력 필드에 "?" 아이콘, 첫 방문 시 가이드 투어
⚠️ 3.0점: FAQ는 있지만 찾기 어려움, 복잡한 기능에 설명 없음 → 사용자가 사용법을 추측해야 함
💡 개선안: 헤더에 "도움말" 메뉴, 각 섹션에 collapsible 안내, 첫 로그인 시 인터랙티브 튜토리얼
```

## 📊 점수 산출 공식

### 기존 (10개 항목)
```
편의성 = (Q1 + Q2 + Q3 + Q4 + Q5 + Q6) / 6
디자인 = (Q7 + Q8 + Q9 + Q10) / 4
종합 = (편의성 + 디자인) / 2
```

### 확장 (25개 항목)
```
각 Nielsen 원칙별 점수 = 해당 세부 항목들의 가중 평균
최종 종합 점수 = (Nielsen 1~10 점수 합) / 10
```

**가중치 예시**:
- 1️⃣ 시스템 상태 가시성: 10%
- 2️⃣ 현실 세계 일치: 12%
- 3️⃣ 사용자 제어: 11%
- 4️⃣ 일관성: 10%
- 5️⃣ 오류 예방: 12%
- 6️⃣ 인식: 10%
- 7️⃣ 효율성: 8%
- 8️⃣ 미니멀 디자인: 10%
- 9️⃣ 오류 복구: 12%
- 🔟 도움말: 5%

## 🎯 실전 적용 예시

### 사례: "고용노동부 대표누리집" (종합 4.47점)

**기존 평가**:
- Q4 메뉴이동: 4.53점 → "메뉴 이동이 쉬웠나요?"

**확장 평가**:
- **N3.1** 뒤로가기: 4.8점 → ✅ 모든 페이지에 "이전" 버튼 명확
- **N3.2** 임시저장: 3.2점 → ⚠️ 긴 신청서에서 임시저장 없음
- **N7.2** 즐겨찾기: 4.5점 → ✅ "자주 찾는 메뉴" 기능 제공
- **N7.3** 고급 검색: 4.7점 → ✅ 날짜/카테고리 필터 우수

**종합 진단**:
```
✅ 강점: 뒤로가기 명확, 즐겨찾기 기능 우수, 검색 필터 다양
⚠️ 약점: 긴 신청서에서 임시저장 부재 → 입력 중 이탈 위험
💡 개선 시: 3.2점 → 4.5점 (임시저장 추가)
💡 예상 종합 점수: 4.47점 → 4.62점 (+0.15점)
```
