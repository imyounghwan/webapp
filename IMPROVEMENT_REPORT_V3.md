## 1. 개선 개요

### 배경
100개 사이트 대규모 검증 결과, 현재 Nielsen 25개 항목 평가 체계에서 **3가지 구조적 문제**가 발견되었습니다:

1. **중복 평가**: 3개 쌍이 100% 동일 (N=56, P<0.001)
2. **검색 의존**: 단일 기능이 3개 항목에 9점 영향
3. **검색 오탐지**: 87.5% 사이트에서 검색 기능 탐지 실패

### 목표
- 논리 타당성: **60% → 90%**
- 독립 항목: **21개 → 22개** (중복 제거 후)
- 검색 탐지율: **12.5% → 70-80%**
- 점수 세밀도: **2단계 → 7단계**

## 2. 5대 핵심 개선

### ✅ 개선 1: 검색 탐지 로직 강화

**문제점**:
```typescript
// 기존 (2가지 패턴만)
const searchExists = 
  /type\s*=\s*["']search["']/i.test(html) || 
  /role\s*=\s*["']search["']/i.test(html)
```
- 결과: **87.5% 오탐지** (49/56 사이트)

**개선**:
```typescript
// 개선 (10가지 패턴)
const searchExists = 
  // HTML5 표준
  /type\s*=\s*["']search["']/i.test(html) ||
  /role\s*=\s*["']search["']/i.test(html) ||
  // name 속성
  /name\s*=\s*["'](search|query|keyword|searchWord|q|kwd)[^"']*["']/i.test(html) ||
  // class/id
  /(class|id)\s*=\s*["'][^"']*search[^"']*["']/i.test(html) ||
  // placeholder
  /placeholder\s*=\s*["'][^"']*검색[^"']*["']/i.test(html) ||
  /placeholder\s*=\s*["'][^"']*search[^"']*["']/i.test(html) ||
  // 버튼/링크
  /<button[^>]*>[^<]*검색[^<]*<\/button>/i.test(html) ||
  /<a[^>]*href\s*=\s*["'][^"']*(search|\/search\.)[^"']*["'][^>]*>[^<]*검색[^<]*<\/a>/i.test(html) ||
  // 근처 텍스트
  /<input[^>]*[^>]*>[\s\S]{0,100}검색/i.test(html)
```
- 예상: **70-80% 탐지율**

### ✅ 개선 2: 중복 항목 제거 (25개 → 22개)

| 제거 항목 | 이유 | 검증 |
|----------|------|------|
| **N3.2 나가기 경로** | N1.1 현재 위치와 100% 중복 (Breadcrumb) | N=56, 100% 일치 |
| **N9.1 오류 메시지** | N5.1 입력 검증과 100% 중복 (폼 검증) | N=56, 100% 일치 |
| **N9.3 오류 예방 정보** | N5.3 제약 조건과 100% 중복 (레이블) | N=56, 100% 일치 |

**효과**: 
- 실제 독립 항목: **21개 → 22개**
- 평가 왜곡 제거

### ✅ 개선 3: 검색 의존 항목 교체/재정의

**제거/교체된 항목**:
| 기존 항목 | 문제 | 개선 항목 |
|----------|------|----------|
| **N6.1 보이는 옵션** | 검색창 의존 | ❌ 제거 |
| **N7.1 단축키** | 검색 = 단축키 (논리 비약) | ✅ N7.1 빠른 접근 (재정의) |
| **N10.1 도움말 접근** | 검색 = 도움말 (논리 비약) | ✅ N10.1 도움말 가시성 (재정의) |

**새 항목 추가**:
- **N7.3 검색/필터 기능**: 검색을 독립 항목으로 평가 (의존 해소)
- **N9.2 복구 지원**: 오류 복구 메커니즘 강화
- **N9.4 오류 안내**: 오류 발생 시 안내 강화

**효과**:
- 검색 기능이 **1개 항목(N7.3)**에만 영향 (기존 3개에서 감소)
- 검색 없는 사이트도 공정한 평가

### ✅ 개선 4: 점수 체계 세밀화 (2단계 → 7단계)

**기존 (양극화)**:
```
2.0 (없음) ────────────────── 5.0 (있음)
           극단적 차이
```

**개선 (7단계 세밀화)**:
```
2.0 (매우 부족)
2.5 (부족)
3.0 (보통 이하)
3.5 (보통)
4.0 (양호)
4.5 (우수)
5.0 (매우 우수)
```

**구현**:
```typescript
const calculateScore = (baseScore: number, adjustment: number): number => {
  const score = Math.max(2.0, Math.min(5.0, baseScore + adjustment))
  // 7단계로 라운딩
  return Math.round(score * 2) / 2
}
```

**효과**:
- 더 정밀한 변별력
- 중간 점수대 사이트 정확히 평가
